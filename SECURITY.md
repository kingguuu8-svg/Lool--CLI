# Security Policy

PocketCLI is designed for self-hosted use, so security issues matter.

## Supported scope

Please report security issues that affect:

- token protection for public access
- VPS tunnel setup
- Android handling of saved connection settings
- server-side session isolation
- accidental exposure of local secrets or files

## How to report

Do not open a public issue for a sensitive vulnerability.

Instead, contact the maintainer privately first and include:

- a short summary
- steps to reproduce
- impact
- any suggested fix or mitigation

If private contact is not available yet, open a minimal public issue without exploit details and ask for a secure channel.

## Response goals

- confirm receipt
- reproduce the issue
- ship a fix or mitigation
- disclose safely after a patch is ready when appropriate
